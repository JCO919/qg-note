# GPIO 与阻塞延时函数

## 一、实验目标

1. 学会编写阻塞延时函数
2. 理解延时参数计算逻辑
3. 掌握GPIO输出控制方法
4. 能通过修改延时实现LED闪烁速度变化

## 二、GPIO基础原理

GPIO即通用输入输出口，本质作用：

控制引脚输出高低电平

单片机控制LED闪烁原理：

```
输出低电平 → LED亮
输出高电平 → LED灭
```

（实际取决于电路接法）

## 三、阻塞延时函数原理

### 什么是阻塞延时？

阻塞延时就是：

程序在延时期间停止执行任何其它任务

CPU会一直执行空循环来消耗时间。

### 延时函数标准写法

```
void delay(unsigned int t)
{
    unsigned int i,j;
    for(i=0;i<t;i++)
        for(j=0;j<110;j++);
}
```

### 执行过程解析

运行流程：

```
进入函数
→ 双重for循环
→ 消耗CPU时间
→ 时间到 → 返回
```

延时长度取决于：

```
循环次数 × 单次循环耗时
```

## 四、延时参数计算逻辑（重点理解）

假设：

```
晶振 = 12MHz
机器周期 = 1μs
```

内层循环一次大约耗时：

```
≈ 1μs × 指令周期数
```

经验近似公式：

```
延时时间 ≈ t × 1ms
```

所以：

| t值  | 实际延时 |
| ---- | -------- |
| 1    | 约1ms    |
| 500  | 约0.5秒  |
| 1000 | 约1秒    |



## 五、LED闪烁程序

### 实验代码

```
#include <reg52.h>

void delay(unsigned int t)
{
    unsigned int i,j;
    for(i=0;i<t;i++)
        for(j=0;j<110;j++);
}

void main()
{
    while(1)
    {
        P1 = 0x00;
        delay(500);

        P1 = 0xff;
        delay(500);
    }
}
```

### 程序逻辑流程

```
亮灯 → 延时 → 灭灯 → 延时 → 循环
```
